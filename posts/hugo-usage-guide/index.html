Hugo 使用指南：从零开始到日常写作

结合 Modern Archivist 主题，详细说明 Hugo 的项目结构、内容管理、命令行操作、Front Matter 写法、模板机制和部署流程。

本文以当前站点为例，系统讲解 Hugo 的核心概念和日常使用方法。所有路径和配置均基于本站实际结构。

项目结构总览

当前站点的目录结构如下：

myblog/
├── hugo.toml                  # 站点主配置文件
├── content/                   # 所有内容（文章、页面）
│   ├── _index.md              # 首页内容
│   ├── about.md               # About 页面
│   ├── archives.md            # 归档页面
│   └── posts/                 # 博客文章目录
│       ├── _index.md          # 文章列表页内容
│       └── my-article.md      # 一篇文章
├── static/                    # 静态资源（图片、文件等）
│   └── images/
│       └── avatar.jpg
├── themes/
│   └── modern-archivist/      # 当前使用的主题
│       ├── layouts/           # 页面模板
│       ├── static/            # 主题静态资源
│       └── theme.toml
└── public/                    # 构建输出（自动生成，不要手动编辑）

各目录的作用

目录 作用 你需要改它吗
content/ 存放所有 Markdown 内容 经常——写文章就是在这里新建 .md
static/ 原样复制到输出的静态文件 偶尔——放图片、PDF 等
themes/ 主题模板和样式 偶尔——调整布局或样式时
public/ 构建产物 永远不要——这是自动生成的
data/ 数据文件（JSON/YAML/TOML） 按需——目前未使用
assets/ 需要 Hugo Pipes 处理的资源 按需——目前未使用

常用命令

启动本地开发服务器

hugo server -D
  • -D 表示包含标记为 draft: true 的草稿文章
  • 启动后访问 http://localhost:1313
  • 修改任何文件后浏览器会自动刷新
  • 按 Ctrl+C 停止

如果端口被占用，可以指定其他端口：

hugo server -D --port 1314

构建站点

hugo

生成的静态文件在 public/ 目录。加上 --minify 可以压缩输出：

hugo --minify

新建文章

hugo new posts/my-new-article.md

Hugo 会根据 archetypes/default.md 模板自动生成包含 Front Matter 的文件。但你也可以直接手动创建文件——效果完全相同。

清理缓存

hugo --gc

写一篇文章

第一步：创建 Markdown 文件

在 content/posts/ 下新建一个 .md 文件。文件名会成为 URL 路径的一部分：

  • content/posts/hello-world.md → /posts/hello-world/
  • content/posts/2026-03-28-my-thoughts.md → /posts/2026-03-28-my-thoughts/

建议使用英文短横线命名，简洁且 URL 友好。

第二步：编写 Front Matter

每篇文章的开头必须有 Front Matter（元数据）。本主题支持 YAML 格式（--- 包围）：

---
title: "文章标题"
date: 2026-03-28
description: "文章摘要，会显示在首页列表和 SEO meta 中"
tags:
  - "标签A"
  - "标签B"
draft: false
---

Front Matter 各字段说明

字段 必填 说明
title 是 文章标题，显示在页面顶部和列表中
date 是 发布日期，格式 YYYY-MM-DD，决定排序顺序
description 推荐 文章摘要。如果不填，列表页会自动截取正文前 160 字
tags 推荐 标签数组，显示为文章顶部和列表中的小标签
draft 否 设为 true 则只在 hugo server -D 时可见，hugo 构建时跳过

第三步：撰写正文

Front Matter 下方用标准 Markdown 撰写正文。本主题支持以下扩展语法：

语法 说明
**粗体** 粗体
*斜体* 斜体
~~删除线~~ 删除线
`行内代码` 行内代码
$E=mc^2$ 行内数学公式（KaTeX）
$$...$$ 块级数学公式（KaTeX）
[^1] / [^1]: 内容 脚注
- [x] 已完成 任务列表

标题层级约定

文章标题已由 Front Matter 的 title 渲染为 H1。因此正文中从 H2 (##) 开始：

## 这是章节标题（H2）

正文内容...

### 这是小节标题（H3）

更详细的内容...

#### 补充说明（H4）

侧边栏的**目录（TOC）**会自动收集 H2 和 H3 级别的标题。这由 hugo.toml 中的配置决定：

[markup.tableOfContents]
  startLevel = 2    # 从 H2 开始
  endLevel = 3      # 到 H3 结束

插入图片

把图片放在 static/images/ 目录下，然后在文章中引用：

![图片说明](/images/my-photo.png)

例如放了一张 static/images/diagram.png，引用方式为：

![架构图](/images/diagram.png)

代码块

使用三个反引号加语言名：

```python
print("Hello, Hugo!")
```

支持的语言高亮包括：python、go、javascript、bash、rust、java、sql、toml、yaml、json、html、css 等。当前高亮主题为 Dracula，可在 hugo.toml 中更改。

数学公式

行内公式用单个 $ 包围，块级公式用 $$ 包围：

行内：$E = mc^2$

块级：

$$\int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}$$

渲染效果：$E = mc^2$

$$\int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}$$

管理 About 页

About 页比较特殊——它同时使用 Front Matter 中的结构化数据和 Markdown 正文。

文件位置

content/about.md

Front Matter 结构

---
title: "About"
layout: "about"           # 必须保留，指定使用 About 专用模板

# ---- 个人信息 ----
name: "你的名字"
role: "你的头衔 / 一句话介绍"
avatar: "/images/avatar.jpg"
email: "you@example.com"

# ---- 社交链接 ----
social:
  twitter: "https://twitter.com/..."
  github: "https://github.com/..."
  linkedin: "https://linkedin.com/..."

# ---- 研究兴趣 ----
research_interests:
  - "方向一"
  - "方向二"
  - "方向三"

# ---- 教育经历 ----
education:
  - degree: "博士学位名称"
    school: "大学名称, 年份"
  - degree: "硕士学位名称"
    school: "大学名称, 年份"

# ---- 发表成果 ----
publications:
  - year: "2024"
    type: "JOURNAL"
    title: "论文标题"
    description: "发表在 *期刊名*。简要描述。"
    tags:
      - "Peer Reviewed"
      - "DOI: 10.xxxx/xxxxx"
  - year: "2023"
    type: "CONFERENCE"
    title: "会议论文标题"
    description: "简要描述"
    tags:
      - "Oral Presentation"

cv_url: "https://链接到你的完整简历"
inquiries_text: "联系说明文字，显示在页面底部。"
---

这里写个人简介的正文，支持完整的 Markdown 语法。
会渲染在 About 页的 Introduction 区域。

关键注意事项

  1. layout: "about" 不能删除——没有它 Hugo 会使用默认的 single.html 模板，About 页的特殊布局就没了
  2. avatar 字段指向 static/ 下的图片。把头像放在 static/images/avatar.jpg 即可
  3. publications 数组中的 description 支持 Markdown 语法（如 *斜体* 表示期刊名）
  4. 添加更多 education 或 publications 条目只需按格式追加即可

管理归档页

归档页自动按年份分组展示所有文章，无需手动维护。

content/archives.md

它的 Front Matter 非常简单：

---
title: "Archives"
layout: "archives"
description: "A chronological index of all published writing."
---

只要你在 content/posts/ 下持续创建文章，归档页会自动更新。

导航菜单管理

导航栏的菜单项在 hugo.toml 中配置：

[[menu.main]]
  name = 'Home'
  url = '/'
  weight = 1

[[menu.main]]
  name = 'Archives'
  url = '/archives/'
  weight = 2

[[menu.main]]
  name = 'About'
  url = '/about/'
  weight = 3

添加新菜单项

例如添加一个「Projects」页面：

第一步：在 hugo.toml 增加菜单项：

[[menu.main]]
  name = 'Projects'
  url = '/projects/'
  weight = 4

第二步：创建对应的内容页 content/projects.md：

---
title: "Projects"
---

这里列出你的项目...

此时它会使用默认的 single.html 模板渲染。如果你需要自定义布局，可以指定 layout 并创建对应模板。

草稿工作流

Hugo 有一个非常实用的草稿机制：

---
title: "还没写完的文章"
draft: true
---
  • draft: true 的文章在 hugo 构建时不会生成
  • 只在 hugo server -D 模式下本地可见
  • 写完后把 draft: true 改为 draft: false（或直接删掉这行）即可发布

日常写作流程

1. 新建文件       →  content/posts/new-article.md
2. draft: true    →  边写边在本地预览
3. hugo server -D →  浏览器实时查看效果
4. 写完改为 false  →  准备发布
5. hugo --minify  →  构建到 public/
6. 部署 public/   →  上线

静态资源管理

所有放在 static/ 目录下的文件会被原样复制到 public/，路径对应如下：

文件系统路径 站点 URL
static/images/avatar.jpg /images/avatar.jpg
static/files/paper.pdf /files/paper.pdf
static/favicon.ico /favicon.ico

模板系统简述

了解 Hugo 的模板查找规则有助于你理解为什么页面长成那个样子。

模板查找优先级

Hugo 渲染一个页面时，会按优先级查找模板：

  1. layouts/ 根目录（项目级覆盖，优先级最高）
  2. themes/modern-archivist/layouts/（主题提供的模板）

也就是说：如果你想修改某个模板但不想改主题源码，可以把主题中的模板文件复制到项目根目录的 layouts/ 下，Hugo 会优先使用它。

示例：想自定义页脚而不改主题：

cp themes/modern-archivist/layouts/partials/footer.html layouts/partials/footer.html

然后编辑 layouts/partials/footer.html——你的修改会生效，且主题源码保持干净。

页面与模板的对应关系

页面 内容文件 使用的模板
首页 content/_index.md layouts/index.html
About content/about.md layouts/_default/about.html（由 layout: "about" 指定）
归档 content/archives.md layouts/_default/archives.html（由 layout: "archives" 指定）
文章列表 content/posts/_index.md layouts/_default/list.html
单篇文章 content/posts/xxx.md layouts/_default/single.html

所有页面共用的部件

每个页面都由 baseof.html 包裹，它引入了三个 partial：

baseof.html
├── partials/head.html     → <head>：CDN、字体、颜色、CSS
├── partials/header.html   → 导航栏
├── {{ block "main" }}     → 各页面独有的内容区
├── partials/footer.html   → 页脚
└── static/js/main.js      → 暗色模式、TOC 交互

部署

方案一：GitHub Pages

# 1. 构建
hugo --minify

# 2. 将 public/ 推送到 GitHub 的 gh-pages 分支
cd public
git init
git add .
git commit -m "deploy"
git remote add origin https://github.com/你的用户名/你的仓库.git
git push -f origin main:gh-pages

更推荐使用 GitHub Actions 自动化，参考 Hugo 官方文档。

方案二：Vercel / Netlify

直接将整个项目仓库连接到 Vercel 或 Netlify，设置构建命令为：

hugo --minify

输出目录设为 public。每次推送代码会自动构建部署。

方案三：自有服务器

hugo --minify
rsync -avz --delete public/ user@server:/var/www/myblog/

常见问题

文章不显示？

检查 Front Matter：

  • date 是否是未来日期？Hugo 默认不渲染未来文章（除非加 -F 参数）
  • draft 是否为 true？构建时需要加 -D 参数或改为 false
  • 文件是否在 content/posts/ 目录下？首页只显示 posts 目录的文章

样式不生效？

  • 确认 hugo.toml 中 theme = 'modern-archivist' 拼写正确
  • 清理缓存重试：hugo server -D --gc --noHTTPCache
  • 浏览器强制刷新：Ctrl+Shift+R 或 Cmd+Shift+R

数学公式显示为源码？

确认 hugo.toml 中有 passthrough 配置：

[markup.goldmark.extensions.passthrough]
  enable = true

这防止 Markdown 解析器将公式中的 _ 等符号错误解析。

如何添加新的独立页面？

  1. 在 content/ 下创建 .md 文件（如 content/projects.md）
  2. 在 hugo.toml 中添加对应菜单项
  3. 如需自定义布局，在 Front Matter 中设定 layout 并创建对应模板

hugo.toml 全配置参考

baseURL = 'https://你的域名/'
languageCode = 'zh-cn'           # 中文站点改为 zh-cn
title = '站点标题'
theme = 'modern-archivist'

[params]
  description = '站点描述'
  homeTitle = '首页大标题'
  homeSubtitle = '首页副标题'
  rssEnabled = true              # 开启 RSS 订阅

  [params.social]
    github = 'https://github.com/xxx'
    scholar = 'https://scholar.google.com/xxx'
    twitter = 'https://twitter.com/xxx'

# 导航菜单
[[menu.main]]
  name = 'Home'
  url = '/'
  weight = 1

[[menu.main]]
  name = 'Archives'
  url = '/archives/'
  weight = 2

[[menu.main]]
  name = 'About'
  url = '/about/'
  weight = 3

# Markdown 渲染配置
[markup]
  [markup.tableOfContents]
    startLevel = 2
    endLevel = 3
  [markup.goldmark.renderer]
    unsafe = true                 # 允许 HTML
  [markup.goldmark.extensions]
    footnote = true
    strikethrough = true
    taskList = true
    [markup.goldmark.extensions.passthrough]
      enable = true               # 数学公式 passthrough
      [markup.goldmark.extensions.passthrough.delimiters]
        block = [['$$', '$$'], ['\[', '\]']]
        inline = [['$', '$'], ['\(', '\)']]
  [markup.highlight]
    style = 'dracula'             # 代码高亮主题
